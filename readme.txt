README - Proyecto GameTify


Este es un proyecto web llamado GameTify que muestra una página web con información sobre juegos populares utilizando la API de Rawg.io. A continuación, se describen los archivos y componentes clave de este proyecto:

Archivos principales
index.html: Este archivo es la página HTML principal del proyecto. Contiene la estructura básica de la página, incluyendo el encabezado, el banner y el contenedor de juegos. La mayoría del contenido de esta página se carga dinámicamente a través de JavaScript.

style.css: El archivo CSS principal que define los estilos visuales de la página. Contiene reglas de estilo para el encabezado, los elementos de navegación, el banner, las tarjetas de juego y los estilos responsivos.

api.js: El archivo JavaScript que maneja las solicitudes a la API de Rawg.io y la presentación de los juegos en la página. La API se utiliza para obtener datos sobre juegos populares y se muestran en tarjetas de juego en la página.

Estructura de la página
Encabezado (header): El encabezado contiene el logotipo "GameTify", una barra de navegación con enlaces a la página de inicio y un cuadro de búsqueda para buscar juegos.

Banner: La sección de banner muestra un mensaje promocional y un botón de "Unirse ahora" que enlaza a una página externa.

Contenedor de juegos: En esta sección se muestran tarjetas de juego que se llenan dinámicamente utilizando datos de la API de Rawg.io. Cada tarjeta de juego muestra la imagen del juego, el nombre, el precio y un enlace para jugar.

Estilos
Los estilos se definen en el archivo style.css y se aplican a diferentes partes de la página. Los estilos se han ajustado para que la página sea responsive y se adapte a diferentes tamaños de pantalla, incluyendo tabletas y teléfonos móviles.

JavaScript
El archivo api.js contiene el código JavaScript necesario para interactuar con la API de Rawg.io. Se utiliza la función fetchAndDisplayGames para realizar solicitudes a la API, obtener datos sobre juegos populares y mostrarlos en la página.

Media Queries
Se han agregado reglas de media queries en el archivo style.css para garantizar que la página se vea adecuadamente en dispositivos más pequeños, como tabletas y teléfonos. Los estilos se ajustan para ocupar menos espacio en la pantalla y garantizar una experiencia de usuario óptima.
